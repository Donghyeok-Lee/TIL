## Python

> "Life is too short, You need Python."
>
> 파이썬에 대해 생각할 수록 이 문장보다 파이썬을 잘 표현하는 방법은 없는 것같습니다.
>
> 새로운 걸 도전하는 것은 언제나 즐거운 일이지만, 그럴 시간도 아쉽고 부족하게 느껴질 때가 있는데,
>
> 그럴때마다 파이썬은 제가 더 중요한 일에 집중할 수 있게 도와주는 것 같습니다. 

### 동적 언어

* 변수 타입을 선언하도록 강요하지 않음 - 컴파일러 대신 **인터프리터**로 해석

    (보통 정적 언어보다 속도가 느리지만, 인터프리터가 최적화되며 속도는 향상되는 중)

* **컴파일러 VS 인터프리터**

    * **컴파일러**
        * Runtime 이전에 전체 소스코드를 보고 명령어를 수집, 재구성
        * 구동 시간이 오래 걸리지만, 구동된 후에는 매우 빠르게 작동
        * 구동 시에 코드와 함께 시스템으로부터 메모리를 할당 받음 / OS 및 빌드 환경에 종속적
    * **인터프리터**
        * Runtime 이후에 소스코드의 각 행을 연속적으로 분석 및 해석하며 실행
        * 실제 실행 시간이 느린 편 (Runtime에 직접 코드를 구동) / 대신 실시간 디버깅 및 코드 수정 가능
        * 메모리를 필요할 때 할당받아서 사용함



### 객체 지향 언어

* 파이썬의 모든 자료는 **객체**

* **객체 지향 프로그래밍 VS 절차적 프로그래밍**

    * **객체 지향 프로그래밍**
        * 관련된 동작들과 관련된 데이터를 함께 묶어서 관리
    * **절차적 프로그래밍**
        * 절차 지향 프로그래밍은 객체 지향 프로그래밍과 달리 ‘객체’라는 개념이 없음
        * 순서대로 특정 명령어들을 실행하는 부분을 하나로 묶은 ‘함수’의 개념
        * 프로그램 안에서 서로 관련된 동작들만을 묶어서 관리

    | 절차적 프로그래밍                                | 객체 지향 프로그래밍                             |
    | ------------------------------------------------ | ------------------------------------------------ |
    | 프로그램을 만들 때 데이터와 함수를 합칠 수 없다. | 프로그램을 만들 때 데이터와 함수를 합칠 수 있다. |
    | 프로그램: 명령어들을 순서대로 실행               | 프로그램: 객체들이 순서대로 소통하는 과정        |

    **데이터와 동작의 연관성이 높고 이걸 객체라는 단위로 묶는 것이 낫겠다는 생각**이 들면 객체 지향 프로그래밍




### 고수준 언어

* 코드를 읽고, 배우기 쉬우며, 작성하기 쉬움! - **생산성이 매우 뛰어남**
* 복잡한 연산을 한 문장으로 표현할 수 있음
* 변수나 인자의 선언이 필요 없다.
* **배터리 포함 (battery included**
    * 사용자가 사용할 수 있는 라이브러리와 통합 환경이 함께 제공됨



### 가상환경

* 프로젝트에 필요한 리소스(패키지)를 효율적으로 관리하기 위함



### 파이썬 사용 시 주의사항

* 프로그램이 계산 작업을 많이 해야할 때 문제가 생김

    (CPU 바운드 - 수행하는 작업은 CPU 속도에 의해 결정됨)
    
    / 시스템 프로그래밍이나 하드웨어 제어와 같은 매우 복잡하고, 반복 연산은 많은 경우 부적절함





## Django

> "The web framework for perfectionists with deadlines."
>
> "마감에 쫓기는 완벽주의자를 위한 웹 프레임워크"

### 왜 Django인가?

* 사실 그냥 Django를 쓴다니까 사용하기 시작했지만,

    그 이유에 대해서는 이제 어느정도는 이해할 수 있게 된 것 같습니다.

* 가장 중요한 것은 Python 기반 프레임워크이며, 빠른 개발이 가능하다는 점이었습니다.

* 간단히 표현하면, **'원하는 것에 더 집중할 수 있게 해주는 것'**이 큰 장점인 것같습니다.



### Django의 특징

* **Ridiculously fast**

    * 설계부터 완성까지 빠르게 프로그램을 구현할 수 있음

        (사용자 인증, 관리 등을 기본적으로 구현해놓음)

    * 개발속도가 빠르고, 코드 완성도를 높게 유지할 수 있으며, 확장성이 우수함

* **Fully loaded**

    * Web 개발 작업에서 상요할 수 있는 다양한 추가 기능을 포함하고 있음

        (User Authentication, Content Administration, Site Maps, RSS feeds, etc...)

* **Reassuringly secure**

    * 보안과 관련된 부분을 처리하는 것을 도와줌
    * SQL injection / CSRF / XSS / Clickjacking 등
    * 사용자 인증 관련된 보안 방법도 존재

* **Exceedingly scalable**

    * 빠르게 변화하는 수요에 맞추어 확장할 수 있는 다양한 가능성이 존재함



### Django의 단점

* 모바일 컴퓨팅에 취약함 (파이썬의 특징)
* 세세한 설정 변경이나 자세한 조정이 어려움





## Vue

> 프론트엔드 프레임워크 / 배우기 쉬움
>
> 리액트와 앵귤러의 장점을 가져온 더욱 빠르고 가벼운 프레임워크

### Vue 특징

* MVVM 패턴 (Model - View - ViewModel)

    화면의 요소들을 제어하는 코드와 데이터 제어 로직을 분리

    * 뷰 : 사용자에게 보이는 화면
    * 모델 : 데이터 저장
    * 뷰 모델 : 뷰와 모델의 중간 영역 / 돔 리스너와 데이터 바인딩을 제공

* 컴포넌트 기반의 프레임 워크

### Vue 장점

* **뛰어난 런타임 퍼포먼스 (빠름)**

    * 뛰어난 렌더링 성능

        * 필요한 DOM 조작 수를 최소화 - 가상의  DOM 추상화 사용

            * 기능 하나를 바꿀 때 기존에는 컴포넌트의 어느 위치에서, 무슨 기능을 바꿀지를 하나씩 다 써줌

                (렌더링 속도도 느려지고, 직관성이 떨어짐)

            * 가상 돔의 경우 복잡한 세부 DOM 트리구조를 적지 않아도 원하는 위치를 쉽게 찾기 가능

        * DOM 조작에 가능한 적은 오버헤드만 가함 - 상태가 없고 인스턴스가 없는 컴포넌트를 제공

    * 갱신 성능

        * 컴포넌트의 종속성이 렌더링 중 자동으로 추적됨

            (시스템이 다시 렌더링해야하는 컴포넌트를 정확히 알고 있음)

* **유연성과 모듈성(확장 가능성)**

    * 다양한 프로젝트에서 보다 적합하게 적용 가능
    * 강력한 라우팅 솔루션 제공 / 상태 관리 솔루션 Vuex 존재
    * CLU 프로젝트 생성기를 제공

* **데이터 바인딩**

    * 컴포넌트 간에 단방향의 데이터 흐름 사용으로 데이터 흐름을 쉽게 파악 가능
    * 양방향 데이터 바인딩도 가능

* 작은 단위의 컴포넌트로 웹 페이지를 구성 가능 - 유지보수와 재사용성 향상

    * 단일 파일 컴포넌트 (HTML, CSS, 자바스크립트가 한 파일에 존재 / 캡슐화 가능)

* 다양한 모듈이 존재

    * Vuex : 중앙 집중식 상태 관리
    * Vue-router : Single Page Application
    * Vue-resource : HTTP, AJAX
    * Vue-devtool 등..
