# Web

## HTML

> **HTML (Hyper Text Markup Language)**
>
> * **Hyper Text** : 각 문서 내부마다 다른 문서에 연결되는 링크를 가지고 있는 형태
> * **Markup** : 그냥 있는 문서들을 큰 제목, 문단 등으로 나누어서 볼 수 있도록 문서의 구조를 만드는 작업
>
> 웹 페이지를 작성하기 위한(구조를 잡기 위한) 언어

### 시맨틱 태그

* HTML5에서 의미론적 요소를 담은 태그의 등장 / 컨텐츠의 의미를 설명하는 태그
* 대표적인 시맨틱 태그의 예시
    * header : 머리말, 제목 등
    * nav : 사이트 간에 서로 연결하는 링크
    * aside : 사이드에 위치하는 공간
    * section : 여러 중심 내용을 감싸는 공간
    * article : 독립적인 글
    * footer : 가장 아래 있는 공간, 회사소개, 저작권, 약관 등
* 개발자 및 사용자 뿐만 아니라 검색 엔진 등에 의미있는 정보의 그룹을 태그로 표현
* 단순히 구역을 나누는 것뿐만 아니라 의미를 가지는 태그를 활용하기 위한 노력





## CSS

> 마크업 언어로 작성된 문서가 실제로 웹사이트에 표현되는 방법을 정해주는 언어
>
> HTML은 문서의 구조화(Markup) / CSS는 스타일 지정

### CSS 정의 방법

* 인라인 : 해당 태그에 직접 style 속성
* 내부 참조 : HTML파일 내에 `<style>`태그 지정
* 외부 참조 : 외부 CSS 파일을 `<head>`내 `<link>`를 통해 불러오기



### CSS 상속

CSS는 상속을 통해 부모 요소의 속성을 **모두 자식에게 상속하지는 않음**

* 속성(property) 중에는 상속이 되는 것과 되지 않는 것들이 있다.
* 상속되는 것 - text 관련 요소 등 (스타일링 관련)
* 상속안되는 것 - Box model 관련 / position 관련 요소 등

MDN 문서에서 확인 가능 / MDN margin : margin과 관련된 속성에 관한 문서에서 상속 여부 확인 가능



### CSS 선택자

> HTML 문서에서 특정한 요소를 선택하여 스타일링하기 위해서 필요

* **기초 선택자**
    * 클래스 (.) / Id (#) / 요소 등
* **고급 선택자**
    * 자손 선택자 ( ), 직계 자손 선택자 (>)
    * 형제 (~), 인접형제 선택자 (+), 전체 선택 (*)
* 의사클래스(pseudo class) / 가상 클래스
* 링크, 동적 의사 클래스
    * 구조적 의사 클래스
    * `p:first-child` / `p:last-child` / `p:nth-child(num)`
        * `p:not(:last-child)` : p 요소 중 마지막 자식이 아닌 경우
        * `p:hover` : 마우스가 p 태그에 올라갔을 때

**CSS 적용 우선순위**

1. **중요도 (Importance) - 사용시 주의** / `!important` (외부 라이브러리 사용 시)
2. 우선 순위 (Specificity)
    * 인라인 / id 선택자 / class 선택자 / 요소 선택자
3. 소스 순서

## 쿠키 VS 세션

* 쿠키(Cookie)
    * 클라리언트의 상태 정보를 로컬에 저장했다가 참조
    * 유효 시간 동안 인증을 유지시켜줌
* 세션(Session)
    * 클라이언트(브라우저)와 서버(웹사이트) 사이의 `state`를 유지시키는 기술
    * 브라우저를 통해서 서버에 접속한 이후 브라우저를 종료하기 전까지 상태(state)를 유지한다.
    * Django는 사용자가 로그인을 하면서 특정한 session id를 발급 -> 클라이언트가 다시 서버에 접속하면 해당 session id가 서버에 포함되어있는 session id와 비교하여 사용자를 식별
* 쿠키 vs 세션
    * 쿠키 : 클라이언트(로컬)에 파일로 저장됨
    * 세션 : 서버에 저장됨 (이때 session id는 쿠키 형태로 클라이언트 로컬에 저장됨)

* 캐시
    * 정보를 가져오는데 비교적 큰 비용이 드는 데이터를 한번 가져온 뒤에 임시로 저장
